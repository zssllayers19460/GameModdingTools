Model Submesh Tool

Split complex models into editable components for detailed modification

What is Model Submesh Tool?

Model Submesh Tool is an advanced utility that splits Saints Row 2 models into their component parts (submeshes). This allows you to edit individual pieces of a model - like a character's shirt separate from their pants, or a car's body separate from its wheels.

Step 1: Understanding Submeshes

Complex models are composed of multiple submeshes:

  • Character Models: Head, torso, arms, legs, hands, feet
  • Vehicle Models: Body, wheels, windows, doors, bumpers
  • Weapon Models: Handle, barrel, magazine, sights
  • Each submesh can have its own: Material, texture, physics, animations

Step 2: Download and Setup

Get the latest version of Model Submesh Tool:

Download: cmesh2obj_0-2_submeshes.rar
Source: saintsrowmods.com/forum/attachments/cmesh2obj_0-2_submeshes-rar.27310

Extract to the same folder as the regular Model Export Tool for best results.

Step 3: Splitting a Model into Submeshes

Extract individual submeshes from a model file:

# Split model into separate submesh files
submesh_tool.exe "model.cmesh_pc" --split

# This creates files like:
- model_submesh_0.obj (first component)
- model_submesh_1.obj (second component)
- model_submesh_2.obj (third component)

# You'll also get a .txt file listing submesh information

Step 4: Analyzing Submesh Data

The tool creates a .txt file with important information:

Example submesh_info.txt:
Submesh 0: Body - 2456 vertices, Material: skin
Submesh 1: Head - 1890 vertices, Material: head_tex
Submesh 2: Hair - 1200 vertices, Material: hair_tex
Submesh 3: Pants - 2340 vertices, Material: denim
Submesh 4: Shirt - 2100 vertices, Material: cotton

Use this to identify which submesh to edit.

Step 5: Editing Individual Submeshes

Edit only the parts you want to change:

  1. Import a single submesh OBJ file into Blender
  2. Make your modifications to that specific component
  3. Export back to OBJ format (maintaining vertex count)
  4. Rename to match original submesh filename
  5. Process with other submeshes you want to keep original
💡 Pro Tip: Keep a backup of all submesh files before editing.

Step 6: Recombining Submeshes

After editing, combine submeshes back into a complete model:

# Recombine edited submeshes
submesh_tool.exe "model.cmesh_pc" --combine

# Or combine specific submesh files
submesh_tool.exe "edited_submesh_0.obj" "original_submesh_1.obj" "edited_submesh_2.obj" -o "new_model.cmesh_pc"

Step 7: Working with Textures

Each submesh uses specific textures:

  • Note the material names from the info file
  • Use PEG Tool to edit the corresponding textures
  • Ensure UV mapping coordinates remain intact after editing
  • Test texture alignment on individual submeshes

Step 8: Advanced Submesh Operations

More complex modifications you can perform:

# Swap submeshes between different models
submesh_tool.exe "car_body.obj" "truck_wheels.obj" -o "custom_vehicle.cmesh_pc"

# Add new submeshes to a model
submesh_tool.exe "character.cmesh_pc" "new_hat.obj" --append

# Remove submeshes from a model
submesh_tool.exe "character.cmesh_pc" --remove 3
(removes submesh index 3)

Common Submesh Mods

  • Outfit Combinations: Mix different clothing pieces from multiple characters
  • Vehicle Customization: Swap wheels, add spoilers, combine car parts
  • Weapon Attachments: Add scopes, silencers, or custom accessories
  • Character Accessories: Add hats, glasses, jewelry to existing models

Performance Considerations

  • Keep total vertex count under game engine limits (~10k per model)
  • Minimize the number of submeshes for better performance
  • Use LOD (Level of Detail) versions for distant objects
  • Test thoroughly after combining submeshes

Troubleshooting

  • Missing textures on specific parts: Check material assignments
  • Model parts misaligned: Verify submesh positioning in 3D space
  • Animation issues: Ensure skeleton bone weights are preserved
  • Combined model won't load: Check submesh order and vertex counts
⚙️ Advanced Workflow Tips:
  • Create a spreadsheet tracking submesh indices and their purposes
  • Use version control for complex multi-submesh projects
  • Test each submesh individually before combining
  • Document your modifications for future reference
