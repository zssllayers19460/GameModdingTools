What is the PEG Tool?

The Peg Tool (specifically sr2pegtool.exe) is a community-developed utility that specializes in extracting and editing .peg_pc and .g_peg_pc texture container files used by Saints Row 2. This powerful tool is essential for any cosmetic modding—allowing you to access, modify, and customize virtually any in-game texture including weapon skins, character clothing, vehicle paint jobs, and environmental textures.

Developed and maintained by the Saints Row modding community, the PEG Tool bridges the gap between the game's proprietary texture containers and standard image formats you can edit in Photoshop, GIMP, or other image editors. It supports the full texture editing workflow including batch processing, format conversion, and even texture animation creation.

Beginner's Workflow Guide

Here's a step-by-step guide for your first PEG Tool project:

  1. Organize your workspace: Create a folder structure (Original_PEG, Extracted, Custom_Textures, Output)
  2. Load the PEG file: Open sr2pegtool.exe and load your source .peg_pc file
  3. Preview textures: Browse and preview all textures to find what you want to edit
  4. Extract textures: Export textures to PNG or TGA format. Recommended: PNG for quality, TGA for compatibility
  5. Edit in Photoshop/GIMP: Make your changes to the extracted texture files
  6. Import back: Use the PEG Tool to import your edited textures back into the container
  7. Export final PEG: Export as .peg_pc for use in-game, or .g_peg_pc for compression
  8. Test in-game: Place your custom PEG file in the game's mod folder and verify it works
  9. Iterate: If textures don't look right, go back to step 5 and refine

💡 Pro Workflow Tip

Always keep backups of your original PEG files. When batch processing, test with a single texture file first to ensure your workflow is correct before processing dozens of files. This saves time and prevents mistakes.

Pro Tips & Tricks

  • Preserve Aspect Ratios: When editing textures, maintain the original dimensions to avoid stretching or distortion
  • Use Layers: In Photoshop/GIMP, work non-destructively with layers. Keep originals separate from edits
  • Test Compression: Some textures work better with ZLIB compression; others with no compression. Test both
  • Mipmaps: The PEG Tool can auto-generate mipmaps. Understanding mipmaps improves texture quality at different viewing distances
  • Animation Timing: For animations, plan frame duration carefully. 50ms per frame = 20 FPS (smooth for most effects)
  • Color Space: Always export from your editor in sRGB color space to maintain color accuracy in-game
  • Quality vs Size: Higher quality = larger file size. Balance visual quality with performance impact
  • Backup Originals: Keep copies of extracted textures before editing to revert if needed

Troubleshooting

Tool Won't Launch

Solution: Run in Windows XP SP3 compatibility mode. Right-click the .exe, Properties → Compatibility → check "Run this program in compatibility mode for Windows XP Service Pack 3"

Textures Won't Extract

Solution: Make sure the PEG file is not read-only. Also verify it's actually a .peg_pc or .g_peg_pc file (not a corrupted or wrong file format). Check file permissions.

Re-imported Textures Look Wrong

Solution: Verify your edited texture is the exact same dimensions as the original. Also check that you're using the correct color depth (DXT1 vs DXT5). Re-extract and compare originals.

Performance Issues

Solution: For large batch operations, close other applications. Process one PEG file at a time. If it crashes, reduce batch size or process files individually.
