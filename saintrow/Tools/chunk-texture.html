What is the Chunk Texture Tool?

The Chunks Texture Tool (SR2ChunksTextureTool.exe) is a specialized GUI utility developed by Masamaru to address a major limitation in Saints Row 2 texture modding. While the standard PEG Tool can edit individual .g_peg_pc files, Saints Row 2 distributes its textures across hundreds of chunk files, making it nearly impossible to edit them individually.

This tool solves that problem by providing batch processing capabilities for chunk data resources. It allows you to collectively manage and replace textures within internal chunk data, making large-scale texture modding projects feasible and efficient. Now you can easily replace textures across multiple chunk files with a single operation.

"Saints Row 2 has a large amount of texture in the chunk data that divided hundreds, it was impossible to edit it in sr2pegtool. I developed SR2ChunksTextureTool to collectively manage the texture of internal chunk data. Now you can easily replace the textures within each chunk data."

— Masamaru, Creator of Chunk Texture Tool

🖼️ What are Chunk Textures?

Saints Row 2 stores many textures across numerous .g_peg_pc chunk files. Instead of editing each file individually, the Chunk Texture Tool processes all chunks at once, allowing for efficient bulk texture replacement.

Why You Need This Tool

The standard PEG Tool (sr2pegtool.exe) is excellent for editing individual texture files, but Saints Row 2's architecture presents a unique challenge:

  • Hundreds of Chunk Files: The game distributes textures across hundreds of .g_peg_pc files
  • Manual Editing Impossible: Opening and editing each file individually would take hundreds of hours
  • Batch Processing Required: Large-scale texture mods need to replace textures across multiple files simultaneously
  • Collective Management: The Chunk Texture Tool enables you to manage all textures as a collective resource

Texture Replacement Workflow

  1. Prepare Your Textures: Create or edit your custom textures using Photoshop, GIMP, or Paint.NET. Save them in the same format as the originals (usually DDS or PNG).
  2. Organize Files: Keep your custom textures in a dedicated folder with names that match the originals or a mapping file.
  3. Load Chunks: Point the Chunk Texture Tool to your extracted chunk files folder.
  4. Map Textures: The tool will display all textures found. Match your custom textures to the ones you want to replace.
  5. Execute Batch Replace: Run the batch replacement operation — the tool will automatically update all relevant chunk files.
  6. Verify Changes: Check a few files to ensure replacements worked correctly before repacking.
  7. Repack and Test: Use ThomasJepp Tools to repack the modified chunk files into VPP_PC archives and test in-game.

🎨 Important: Match Original Formats

When replacing textures, ensure your custom textures match the original's dimensions and compression format. Using mismatched formats can cause visual artifacts or game crashes. Extract a reference texture first to check the original specifications.

Pro Tips for Texture Modding

📊 Start Small

Test your workflow with a few textures first before processing hundreds of files. This helps identify any issues early.

💾 Always Backup

Keep backups of your original chunk files. You'll need them to revert changes if something goes wrong.

📁 Organize by Category

Group textures by category (characters, vehicles, environment) to make batch replacements easier to manage.

🎨 Use Mipmaps

Generate mipmaps for your textures to ensure they look good at all distances. The Chunk Texture Tool can auto-generate them.

🔍 Reference Originals

Always keep an extracted reference texture to compare dimensions, format, and color depth before replacement.

⚡ Batch in Stages

Process textures in batches by category rather than all at once. This makes troubleshooting easier.

Troubleshooting

Textures don't appear in-game after replacement

Fix: Verify your custom textures match the original dimensions and format. Check that the chunk files were correctly repacked into VPP_PC archives.

Game crashes when loading modified chunks

Fix: Ensure your textures use the correct compression format (DXT1, DXT3, or DXT5). Extract a reference texture to check the original specs.

Tool can't find chunk files

Fix: Make sure you've extracted the chunk files from VPP_PC archives first using ThomasJepp Tools. Point the tool to the extracted folder containing .g_peg_pc files.

Textures look corrupted or pixelated

Fix: Your custom texture may have mismatched dimensions or mipmaps. Ensure dimensions are powers of two (64, 128, 256, 512, 1024, etc.).

Batch replacement takes too long

Fix: Process textures in smaller batches. The tool is processing hundreds of files — large projects will naturally take time.
