What is ThomasJepp Saints Row Tools?

ThomasJepp Saints Row Tools is a specialized utility designed to unpack and repack Saints Row's proprietary VPP_PC archive files. VPP_PC files are compressed containers that hold the bulk of the game's assets—everything from 3D character models and vehicle meshes to textures, animations, sound effects, dialogue, and gameplay data.

This tool is the foundation of all asset-based modding in Saints Row 2. Without it, modders would have no way to access the game's internal file structures, making cosmetic changes, model swaps, texture replacements, and major gameplay modifications impossible. It essentially bridges the gap between the game's compressed, optimized format and human-readable asset files that can be edited in external applications.

Core Function

Decompresses VPP_PC archives → Extracts individual assets → Allows editing → Repacks modified files back into game format

Key Features & Capabilities

VPP_PC Extraction

Unpacks the main game archive files, giving you access to thousands of individual assets organized by type and location.

Asset Organization

Automatically sorts extracted assets into logical folders (models, textures, animations, etc.) for easy navigation and editing.

Batch Processing

Process multiple VPP_PC files at once, saving time when working with large-scale mods or total conversions.

Asset Repacking

Recompile modified assets back into the game's native VPP_PC format so the game recognizes and loads your changes.

Format Support

Works with all major Saints Row 2 asset types including CMESH (character models), SMESH (static meshes), and PEG (textures).

Customization

Full control over what gets extracted, edited, and repacked—perfect for creating custom content or total overhauls.

Complete Workflow: From Asset to Mod

Here's the step-by-step process for using ThomasJepp Tools to create a complete mod:

1

Locate Game Files

Navigate to your Saints Row 2 installation directory and identify the VPP_PC files you want to extract from. These are typically found in the game's data folder and are named something like packfiles.vpp_pc or similar.

C:\Program Files\Saints Row 2\data\packfiles.vpp_pc
2

Open ThomasJepp Tools

Launch the ThomasJepp Saints Row Tools executable. The interface will present options for extraction, repacking, and asset management. Select the VPP_PC file you want to work with.

Tip: Make a backup of the original VPP_PC file before extracting. This way you can always revert if something goes wrong.

3

Extract Assets

Choose your extraction settings and let the tool unpack the VPP_PC archive. This creates a folder structure containing all the game's assets organized by type. The extraction can take several minutes depending on file size and your system speed.

You'll now have access to:

  • 3D Models: Character skins, vehicles, weapons, environmental objects
  • Textures: Diffuse maps, normal maps, material properties
  • Animations: Character movements, vehicle animations, emotes
  • Audio: Voice lines, sound effects, music
  • Gameplay Data: Mission scripts, item definitions, NPC behavior
4

Identify & Prepare Assets

Browse the extracted folders to find the specific assets you want to modify. This might be:

  • A character model file (CMESH) for skinning
  • A texture file (PEG) for recoloring
  • A vehicle model (CAR_PC) for customization

Copy the assets you want to modify to a working directory outside the extraction folder.

5

Edit Assets with External Tools

Use specialized tools or 3D software to modify your extracted assets:

  • Textures: Use Photoshop, GIMP, or the PEG Tool to edit texture files
  • Models: Export to OBJ using the Model Export Tool, then edit in Blender or 3DS Max
  • Audio: Use audio editing software (Audacity, etc.) for sound modifications
  • Scripts: Edit LUA or XTBL files with a text editor for gameplay changes

Pro Tip: Always work on copies of assets. Keep the originals intact as reference.

6

Replace Modified Assets

Copy your edited assets back into the extracted folder structure, replacing the original files. Make sure you maintain the exact same file paths and names—the game needs to find these files exactly where it expects them.

extracted_folder/characters/player/player_female.cmesh
extracted_folder/textures/vehicles/taxi_paint.peg
7

Repack Assets into VPP_PC

Open ThomasJepp Tools again and select the repack/compile option. Point it to your modified asset folder and let it rebuild the VPP_PC archive with your changes. This creates a new packfile ready for the game to use.

Important: The repacked file must maintain the same name and location as the original for the game to load it. Alternatively, you can create a separate mod folder.

8

Test in Game

Launch Saints Row 2 and verify that your modifications appear correctly. Check for:

  • Visual quality and appearance of textures/models
  • No missing assets or broken references
  • Proper animations and interactions
  • No game crashes or stability issues

If something isn't working, revert to your backup and troubleshoot your modifications.

9

Distribute Your Mod

Once you're satisfied with your mod, package it for distribution. You can share:

  • The modified VPP_PC file directly
  • A mod folder with installation instructions
  • Source files + instructions for other modders to use

Upload to Saints Row Mods to share with the community!

Tips & Best Practices

Always Backup

Keep copies of the original VPP_PC files before extraction. This is your safety net if something breaks.

Document Changes

Keep notes on what you've modified. This helps you troubleshoot and makes it easier to maintain your mod later.

Test Incrementally

Test your mod after each significant change. This helps isolate which modification caused problems.

Check File Paths

Ensure modified assets are in the exact same folder structure. The game won't find files if they're in the wrong location.

Use Version Control

If making major mods, use Git or similar to track changes and revert if needed.

Optimize Textures

Compress textures appropriately. Overly large textures can slow down the game or cause memory issues.

Common Issues & Troubleshooting

Game Crashes on Load

Cause: Corrupted or improperly formatted asset files.

Solution: Verify your edited assets are valid. Re-export and re-edit if necessary. Check that the VPP_PC was properly repacked.

Modified Assets Don't Appear

Cause: Assets in wrong folder path or not properly repacked.

Solution: Double-check file paths match the original. Ensure the VPP_PC is in the correct game directory and named correctly.

Missing Textures (Pink/Magenta)

Cause: Texture references are broken or assets weren't properly linked.

Solution: Verify texture file names match exactly. Re-export and repack the affected assets.

Tool Won't Open VPP_PC

Cause: Unsupported version or corrupted file.

Solution: Restore from backup. Ensure you're using the correct tool version for your game version.

Performance Issues

Cause: Overly complex models or high-resolution textures.

Solution: Optimize asset complexity and texture sizes. Match the original game's quality standards.
