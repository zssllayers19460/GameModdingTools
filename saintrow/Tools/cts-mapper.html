Visualise position data from Saints Row 2's CTS files directly on the game map — see exactly where CD locations, tagging spots, and other world objects are placed.

Map Viewing CTS Files Position Data v2.01

What is CTS Mapper?

CTS Mapper is a visualisation tool created by Masamaru that reads Saints Row 2's .cts (Common Terrain Streamer) files and draws the position data they contain directly onto the game map. Rather than trying to interpret raw coordinate values from a text editor, you can see every tagged location plotted visually in its real in-world position.

"This tool draws on the map the position information described in the cts file. For example, in cds_sr2_city.cts it contains the position which is located CD. Position of the tag has been stored in the tagging_spots.cts as well."

— Masamaru, Saints Row Mods forum

CTS files act as data tables that tell the game where to place a wide range of world objects — from collectible CD locations to graffiti tagging spots and beyond. CTS Mapper makes that hidden position data visible, which is essential for modders who want to understand, edit, or add to the game world.

Core Concept

Load a .cts file → CTS Mapper reads the position tags inside it → each position is plotted as a point on the Stilwater overhead map → you can see every object's exact real-world location at a glance.

Understanding CTS Files

CTS files are found throughout Saints Row 2's data archives and each one stores position data for a specific category of world object. Here are some key examples the tool works with:

File Contains Used For
cds_sr2_city.cts CD collectible positions Finding/modding all CD disc locations across Stilwater
tagging_spots.cts Graffiti tag positions Viewing and repositioning gang tag spray locations
mission spawns Player/NPC spawn coords Editing where missions start and characters appear
activity_spots.cts Activity trigger locations Placing or relocating side activities and mini-games

Before You Start

You'll need to extract the relevant .cts files from the game's VPP_PC archives first. Use ThomasJepp Saints Row Tools to unpack the archives before loading files into CTS Mapper.

Key Features

🗺️ Map Overlay Visualisation

Plots every position entry from a .cts file as a visible marker on an overhead view of Stilwater, giving you instant spatial context for raw coordinate data.

📍 Positional Tag Reading

Reads the internal position tags from CTS files exactly as the game engine does, so what you see in the tool matches what the game uses at runtime.

🎯 CD Location Mapping

Load cds_sr2_city.cts to see all collectible CD disc positions plotted on the map — invaluable for building collection guides or relocating pickups.

🖊️ Tagging Spot Support

Supports tagging_spots.cts to visualise every graffiti tag location across Stilwater's districts, useful for gang territory and activity modding.

📂 Multi-file Compatible

Works with any CTS file that stores positional data, not just the two examples above — experiment with other .cts files from the game's archives.

🔍 Coordinate Reference

Helps you identify the X/Y/Z coordinates of specific locations visually, so you can copy accurate values into other modding tools or script files.

How to Use CTS Mapper

1

Extract Game Archives

Use ThomasJepp Saints Row Tools to unpack the game's VPP_PC files. The CTS files you need are stored inside these archives — you must extract them before CTS Mapper can read them.

2

Locate Your Target CTS File

Browse the extracted files and find the .cts file you want to inspect — for example cds_sr2_city.cts for CD locations or tagging_spots.cts for graffiti tags.

3

Open CTS Mapper & Load the File

Launch CTS Mapper and open your chosen .cts file. The tool will parse the position data and draw each entry as a point on the Stilwater map overlay.

4

Read & Record Positions

Use the map view to identify where objects are placed. Note down specific X/Y/Z coordinates that you want to use or modify in other tools or XML/Lua script files.

5

Edit the CTS File & Repack

Make your changes to the .cts file in a text or hex editor using the coordinates you've gathered. Repack the VPP_PC archive with ThomasJepp Tools and test in-game.

Common Issues

CTS Mapper won't load the file

Cause: File wasn't fully extracted, is corrupted, or is the wrong CTS version.

Fix: Re-extract the file from a clean copy of the game. Make sure you're using a PC version archive, not a console rip.

Map shows no markers / blank map

Cause: The loaded CTS file may not contain positional tags, or they're in a format the tool doesn't recognise.

Fix: Try a known-good file like cds_sr2_city.cts first to confirm the tool is working, then test other files.

Positions look offset or wrong in-game after editing

Cause: Z-axis value may be incorrect, causing spawns to appear underground or floating.

Fix: Cross-reference Z values with known working positions from nearby entries in the same CTS file.

Changes don't appear after repacking

Cause: The edited file wasn't saved correctly before repacking, or the wrong VPP archive was updated.

Fix: Verify the modified .cts file was saved, then repack using ThomasJepp Tools and confirm the output archive is in the correct game directory.
